# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

Static marketing website for the **Lenguaje Claro** Chrome extension — a tool that detects archaic terms, legalese, passive voice, and similar style issues in legal Spanish text within Google Docs.

Deployed on Netlify at <https://extensionlenguajeclaro.com.ar/>.

## Build system

The site uses a **custom Node.js template builder** (`build.mjs`) — no webpack, Vite, or other bundler. Run from the monorepo root:

```bash
npm run build   # one-time build
npm run watch   # rebuild on file changes (polls every 300 ms)
```

**Never edit `index.html` or `legal.html` directly** — they are generated and will be overwritten. Edit the source files instead.

### How it works

1. Loads partials from `src/partials/`
2. Reads templates from `src/pages/`
3. Replaces `{{ variableName }}` placeholders with partial content
4. Writes output files to the `public/` directory (`public/index.html`, `public/legal.html`, `public/support.html`)
5. Prepends a `<!-- generated by build.mjs -->` comment to each output

Missing placeholders throw an error at build time.

## Design system

All colors use **oklch**. CSS custom properties on `:root` define the full palette; dark mode overrides them under `[data-theme="dark"]`. The accent color is terracotta (`--accent`). Never use raw hex or rgb values — always use the design tokens.

Theme is stored in `localStorage` and applied before paint via the inline script in `shared-scripts.html` (prevents FOUC). Applied as `data-theme="light|dark"` on `<html>`.

Typography uses three Google Fonts: **Geist** (sans-serif body), **Fraunces** (serif headings/display), **IBM Plex Mono** (code/labels). Always reference via `var(--font-sans)`, `var(--font-serif)`, `var(--font-mono)`.

Layout uses `.wrap` for max-width centering (`--maxw: 1180px`) and `clamp()` for fluid spacing. Section structure follows `.block` > `.block-header` + content. Key breakpoints: 900 px (hero becomes single-column), 760 px (nav links hide).

## Hero demo (`demo.js`)

The demo widget in `#demo-body` renders a hardcoded legal paragraph with six `.mark` spans. It auto-cycles every 2.6 s, opening a `.popover` on each mark. User click stops the cycle. The "Aplicar cambio" button replaces the mark text with the suggestion inline.

Color per rule kind is defined in `COLOR_BY_KIND` at the top of `demo.js`. To add a new demo example, add a segment object with `{ m, kind, text, from, to, label }` to the `paragraph` array.

5. **Features** (`02`, `#features`) — 2×4 grid of rule cards (`.feat`)
6. **Screenshot** (`03`) — `assets/screenshot.png`
7. **Comparison** (`04`, `#comparacion`) — `.comp-table` with ✓/×/parcial checks
8. **Privacy** (`05`, `#privacidad`) — summary cards + inline policy document (`§1–§6`)
9. **Waitlist CTA** (`#descargar`) — email form (client-side only, no backend)
10. **Footer** (partial: `footer.html`) — brand, links, legal

## Waitlist form

The form (`#waitlist`) has no backend. On submit it shows `#cta-thanks` and disables itself. To wire a real mailing list, add a `fetch()` POST inside the submit handler in `index.template.html`.
